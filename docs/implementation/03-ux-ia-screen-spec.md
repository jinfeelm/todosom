# UX/IA와 화면 명세

## 내비게이션 구조

MVP는 Expo Router 기반 하단 탭 4개로 구성한다.

| 탭 | 경로 예시 | 역할 | MVP 처리 |
| --- | --- | --- | --- |
| 오늘 | `/today` | 투두 작성, 완료, 핵심 씨앗 | 필수 |
| 펫방 | `/pet-room` | 펫 보기, 말 걸기, 결 상태 | 필수 |
| 기록 | `/archive` | 도감, 이전 여정, 진화 기록 | 필수 |
| 상점 | `/store` | 유료 펫, 방, 테마 | placeholder |

첫 실행 사용자는 `/onboarding`으로 보내고, 온보딩 완료 후 `/today`로 이동한다. 이미 온보딩을 완료한 사용자는 앱 실행 시 `/today`로 간다.

## 온보딩

온보딩은 설명보다 행동을 먼저 만든다. 사용자가 첫 씨앗을 심고 완료하면서 제품을 이해해야 한다.

| 단계 | 화면 핵심 | 문구 방향 | 구현 동작 |
| --- | --- | --- | --- |
| 1 | 솜알 등장 | 오늘의 작은 할 일이 이 아이의 결을 만듭니다 | 기본 펫 인스턴스를 `egg`로 준비 |
| 2 | 첫 할 일 입력 | 첫 씨앗을 하나 심어볼까요? | 제목만으로 빠르게 생성 |
| 3 | 첫 완료 | 씨앗 하나가 결이 되었어요 | 완료 이벤트와 첫 결 점수 생성 |
| 4 | 솜몽 부화 | 처음엔 모두 같은 아이예요 | `life_stage`를 `baby`로 변경 |
| 5 | 오늘 화면 진입 | 이제 오늘의 핵심 씨앗을 골라볼까요? | `/today`로 이동 |

### 온보딩 수용 기준

- 5분 안에 첫 완료까지 갈 수 있어야 한다.
- 가입, 결제, 카테고리 세부 설정을 요구하지 않는다.
- 첫 카테고리가 없으면 기본 카테고리 `오늘`과 기본 결 `focus`를 임시로 사용해도 된다.

## 오늘 화면

오늘 화면은 생산성 도구로서 가장 빠르고 명확해야 한다.

```text
[상단]
오늘 날짜
펫의 짧은 상태 문구

[핵심 씨앗 영역]
오늘의 핵심 씨앗 1~3개

[일반 투두 리스트]
미완료 항목
완료 항목 접기

[하단 액션]
+ 씨앗 심기
```

### 오늘 화면 구성 요소

| 요소 | 동작 |
| --- | --- |
| 날짜 헤더 | 오늘 날짜와 짧은 펫 상태를 표시한다 |
| 핵심 씨앗 카드 | `is_core_seed = true`인 오늘 투두를 최대 3개 표시한다 |
| 일반 투두 리스트 | 핵심 씨앗이 아닌 오늘 투두를 표시한다 |
| 완료 버튼 | 탭 즉시 완료 처리, 결 획득 toast 표시 |
| 씨앗 심기 버튼 | 입력 sheet 또는 modal을 연다 |
| 빈 상태 | 첫 씨앗을 하나 심어보자는 문구와 입력 CTA를 보여준다 |

### 투두 입력 필드

| 필드 | MVP | 기본값 |
| --- | --- | --- |
| 제목 | 필수 | 없음 |
| 날짜 | 필수 | 오늘 |
| 카테고리 | 필수 | 최근 사용 카테고리 또는 기본 카테고리 |
| 결 | 필수 | 카테고리의 `gyeol_type` |
| 핵심 씨앗 | 필수 | false |
| 난이도 | 선택 | `normal` |

반복, 메모, 서브태스크는 MVP 입력 화면에 넣지 않는다.

### 핵심 씨앗 선택 규칙

- 오늘 날짜 기준 최대 3개까지 선택 가능하다.
- 3개 초과 시 "오늘 가장 빛나게 만들 씨앗은 세 개면 충분해요"처럼 부드럽게 안내한다.
- 핵심 씨앗은 완료 시 진화 점수 보너스를 받는다.

## 펫방 화면

펫방은 TodoSom의 감정 중심 화면이다. 오늘 화면이 실용의 공간이라면 펫방은 애착의 공간이다.

```text
[상단]
펫 이름 / 생애 단계

[중앙]
고해상도 픽셀 펫
방 배경
상태 이펙트

[하단]
오늘 쌓인 결 요약
말 걸기 / 상태 보기
```

### 펫방 구성 요소

| 요소 | 동작 |
| --- | --- |
| 펫 스프라이트 | 현재 `life_stage`, `primary_gyeol`, `asset_key`에 맞는 이미지 표시 |
| 방 배경 | 기본 방 theme 표시 |
| 말풍선 | 상태 기반 대사 표시 |
| 오늘의 결 요약 | 오늘 획득한 결 타입별 점수 표시 |
| 말 걸기 버튼 | 대사 조건을 평가해 새 대사 표시 |
| 결정 먹이기 버튼 | MVP에서는 완료 후 자동 반영으로 처리하거나 비활성화 |
| 진화 modal | 조건 충족 시 성장기 진화를 보여준다 |

## 기록 화면

기록 화면은 "삭제"가 아니라 "보관"의 감각을 만들어야 한다.

```text
[현재 여정]
솜몽 -> 성장기 형태
주요 결
진화일

[기록의 방]
이전 펫 카드 목록

[상세]
진화 계기
대표 투두
당시 결 분포
```

### 기록 화면 구성 요소

| 요소 | 동작 |
| --- | --- |
| 현재 여정 카드 | 현재 펫의 생애 단계와 주요 결 표시 |
| 이전 여정 목록 | `life_stage = archived` 또는 archived pet instance 표시 |
| 진화 기록 상세 | `evolution_history`와 대표 `gyeol_score_events`를 연결해 표시 |
| 새 여정 CTA | 성장기 이후 사용할 수 있으며 확인 dialog를 띄운다 |

### 새 여정 확인 문구

> 이 아이와의 여정을 기록의 방에 남길까요? 사라지는 것이 아니라, 당신이 지나온 결의 모습으로 기록됩니다.

새 여정은 무료여야 한다. 완전 삭제는 MVP에서는 제공하지 않는다.

## 상점 화면 placeholder

MVP에서 상점은 실제 결제를 하지 않는다. 단, 장기 구조와 BM 윤리를 화면 언어에 반영한다.

| 영역 | 표시 내용 |
| --- | --- |
| 유년기 펫 | 별솜, 잉크솜 등 coming soon 카드 |
| 방 테마 | 비 오는 공부방, 새벽 공부방 등 coming soon 카드 |
| 펫 소품 | 리본, 작은 가방 등 coming soon 카드 |
| 윤리 안내 | 핵심 진화와 기본 여정은 무료로 충분히 즐길 수 있음을 명시 |

## 공통 UX 언어

| 하지 않는다 | 한다 |
| --- | --- |
| 왜 안 했어요? | 오늘은 하나만 다시 밝혀볼까요? |
| 실패했습니다 | 안개가 조금 끼었어요 |
| 점수가 부족합니다 | 몸의 결을 바꾸려면 묵직한 씨앗이 필요해요 |
| 연속 기록이 깨졌습니다 | 다시 돌아온 것도 성장의 일부예요 |
| 지금 사지 않으면 놓칩니다 | 새로운 솜이 찾아왔어요 |

## 기본 화면 상태

각 화면은 최소 4개 상태를 고려한다.

| 상태 | 설명 |
| --- | --- |
| loading | 로컬 DB 초기화 또는 데이터 조회 중 |
| empty | 첫 사용 또는 해당 데이터 없음 |
| ready | 정상 데이터 표시 |
| error | DB 읽기/쓰기 실패, migration 실패 |

에러 문구는 사용자 탓으로 쓰지 않는다. 예: "잠깐 결을 정리하지 못했어요. 다시 열어볼까요?"
