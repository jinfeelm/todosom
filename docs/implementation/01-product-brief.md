# TodoSom 제품 브리프

## 한 문장 정의

TodoSom은 사용자가 직접 만든 할 일을 완료하면 `결`이 쌓이고, 그 결의 성향에 따라 고해상도 픽셀 솜뭉치 펫이 다르게 진화하는 펫 투두 앱이다.

## 브랜드 코어

> 같은 솜뭉치가, 다른 하루를 먹고, 다르게 자란다.

같은 기본 펫에서 시작하지만 사용자의 실제 하루가 펫의 형태를 바꾼다. TodoSom의 차별점은 생산성 결과를 숫자나 그래프가 아니라 애착 가능한 생명체로 번역하는 데 있다.

## 제품 카테고리

| 구분 | 내용 |
| --- | --- |
| 1차 카테고리 | 개인용 투두/루틴 앱 |
| 2차 카테고리 | 펫 육성·수집형 생산성 앱 |
| 감성 카테고리 | 다이어리, 꾸미기, 픽셀 펫, 작은 성취 |
| 장기 확장 | 디지털 펫 IP, 방 꾸미기, 시즌 콘텐츠, 굿즈, 커뮤니티 |

## 핵심 사용자 문제

기존 투두 앱은 할 일이 쌓이고 미완료 항목이 남을수록 도구가 아니라 죄책감의 화면이 되기 쉽다. TodoSom은 사용자의 실패를 벌점으로 기록하지 않고, 다시 돌아와 정리할 수 있는 상태로 표현한다.

| 문제 | 사용자 감정 | TodoSom의 대응 |
| --- | --- | --- |
| 할 일이 많다 | 어디서 시작할지 모르겠다 | 오늘의 핵심 씨앗 1개에서 3개로 줄인다 |
| 완료하지 못했다 | 또 실패했다는 느낌이 든다 | 미완료를 `안개` 또는 재정리 대상으로 표현한다 |
| 앱을 며칠 쉬었다 | 돌아가기 민망하다 | 펫은 죽지 않고 잠들었다가 다시 깨어난다 |
| 통계가 차갑다 | 내 노력이 의미 있게 남지 않는다 | 결과를 펫 진화와 도감으로 바꾼다 |
| 귀여운 앱은 기능이 약하다 | 예쁘지만 오래 쓰지 못한다 | 투두 기본기는 심플하고 빠르게 유지한다 |

## 핵심 루프

TodoSom의 모든 기능은 아래 루프를 강화해야 한다.

```text
할 일 작성
-> 핵심 씨앗 선택
-> 완료
-> 결 획득
-> 펫 반응
-> 진화 기대
-> 기록 또는 새 여정
```

새 기능이 사용자를 투두 완료 전에 다른 곳으로 새게 만들면 후순위다. 새 기능이 완료 감정, 펫 애착, 진화 기대감을 강화하면 채택 후보가 된다.

## 반드시 지킬 5가지

1. 투두 입력은 빠르고 심플해야 한다. 귀여움을 위해 생산성의 기본을 망치지 않는다.
2. 펫은 압박 장치가 아니라 동반자여야 한다. 실패했을 때도 사용자를 기다린다.
3. 무료 유저도 핵심 진화와 기록의 재미를 충분히 경험해야 한다.
4. 유료 상품은 더 강한 펫이 아니라 더 다른 펫, 더 예쁜 방, 더 풍부한 서사를 제공해야 한다.
5. 픽셀 아트는 저해상도 느낌이 아니라 고해상도·고품질 컬러 도트 감성으로 간다.

## 제품 원칙

| 원칙 | 구현 의미 |
| --- | --- |
| 하나의 핵심 루프만 먼저 이긴다 | MVP에는 투두, 결, 펫 반응, 진화, 기록만 필수로 넣는다 |
| 모두 같은 기본 솜에서 시작한다 | 기본 유년기 펫 `솜몽`을 브랜드 대표 캐릭터로 만든다 |
| 진화는 돈이 아니라 행동으로 결정된다 | 결제는 외형과 서사 확장에만 쓰고 진화 성능에는 영향을 주지 않는다 |
| 실패를 벌하지 않는다 | 펫 사망, 강한 패널티, 공격적 리마인더를 금지한다 |
| 투두는 심플하게, 펫은 고급스럽게 | 오늘 화면은 빠르게, 펫방은 정서적으로 풍부하게 만든다 |
| 어뷰징은 막기보다 무력화한다 | 가벼운 완료는 인정하되 진화 영향은 낮춘다 |
| 확장성은 데이터 구조로 먼저 열어둔다 | 방, 인벤토리, 유료 상품은 UI보다 모델 구조를 먼저 준비한다 |

## 메인 타깃

우선 타깃은 10대 후반부터 30대 여성이다. 단, 여성향을 단순히 핑크와 리본으로 해석하지 않는다. 기능성과 감성을 동시에 만족해야 한다.

| 타깃 | 상황 | 욕구 | 제공 가치 |
| --- | --- | --- | --- |
| 10대 후반 | 공부, 수행평가, 덕질, 다이어리 | 작은 성취를 귀엽게 기록하고 싶다 | 공부와 루틴이 펫 진화로 남는다 |
| 20대 | 대학, 취업, 자기계발, 콘텐츠, 운동 | 갓생 압박 없이 꾸준히 하고 싶다 | 완벽보다 복귀와 진행을 인정한다 |
| 30대 | 일, 건강, 집안일, 관계, 회복 | 할 일을 정리하되 부담 없고 싶다 | 오늘의 핵심 씨앗으로 복잡함을 줄인다 |

## 포지셔닝

Todoist처럼 빠르게 적고, Things처럼 군더더기 없이 예쁘고, 다마고치처럼 자꾸 보고 싶고, 디지몬처럼 내가 키운 결과가 다르게 남는 앱.

## 주요 용어

| 용어 | 의미 | 구현 역할 |
| --- | --- | --- |
| 솜알 | 아직 깨어나지 않은 시작 상태 | 온보딩 전 `egg` stage |
| 솜몽 | 모든 사용자의 공통 유년기 기본 펫 | 기본 `pet_species` |
| 씨앗 | 사용자가 만든 할 일 | `todos` |
| 핵심 씨앗 | 오늘 가장 중요한 1개에서 3개 할 일 | `is_core_seed` |
| 결 | 할 일의 성향 에너지 | `GyeolType` |
| 간식결 | 가벼운 완료 보상 | 진화 영향 낮음 |
| 진화결 | 중요한 완료 보상 | 진화 영향 높음 |
| 회복결 | 미룬 일을 재정리하고 완료한 보상 | 복귀 보정 |
| 안개 | 미뤄지거나 정리되지 않은 상태 | 부드러운 리마인더 상태 |
| 기록의 방 | 이전 펫과 여정을 보관하는 곳 | archive screen |
| 새 여정 | 기존 펫을 기록하고 다시 시작하는 행위 | reset 대신 쓰는 UX 언어 |

## 제품이 아닌 것

- 엄격한 업무관리 SaaS가 아니다. 팀 협업, 관리자 감시, 프로젝트 관리는 MVP 범위 밖이다.
- 순수 모바일 게임이 아니다. 게임성은 동기부여 수단이고 본체는 투두 관리다.
- 죄책감 자극 앱이 아니다. 연속 기록이 깨져도 사용자를 벌하지 않는다.
- 가챠 중심 캐릭터 앱이 아니다. 초기 BM은 직접 구매형 펫, 테마, 방 아이템 중심이다.
- AI 상담 앱이 아니다. 초기 펫 대화는 상태 기반 짧은 반응이다.
