# 무료 베타부터 첫 BM까지의 1차 MVP

## 배경

`todosom_whitepaper_v0_1.docx`는 TodoSom이 커져서 수익화되고, 사업체와 팀이 생긴 뒤에도 방향을 잃지 않기 위한 큰그림 문서다. 따라서 백서의 모든 내용을 첫 개발 범위로 오해하면 안 된다.

이 문서에서 말하는 1차 MVP는 "로컬 프로토타입"만이 아니다. 무료 베타로 실제 사용자를 만나고, 첫 BM 상품을 작게 검증하는 순간까지를 하나의 1차 MVP로 본다.

## 1차 MVP 한 문장 정의

무료로 충분히 사랑받을 수 있는 TodoSom 핵심 루프를 출시하고, 사용자가 돈을 내도 납득할 첫 감성 확장 상품 1개를 검증하는 것.

## 1차 MVP 성공 기준

1차 MVP는 다음 조건을 만족하면 완료로 본다.

- 사용자가 무료로 온보딩, 첫 씨앗 완료, 솜몽 부화, 펫방 반응을 경험한다.
- 무료 사용자가 7개 결 기반 성장기 진화와 기록의 방을 경험할 수 있다.
- 베타 사용자의 첫 투두 생성률, 첫 투두 완료률, 펫방 방문률, 말 걸기 클릭률, D1/D7 리텐션을 볼 수 있다.
- 실제 스토어 또는 테스트 배포 채널에서 설치 가능한 앱이 있다.
- 개인정보 처리방침, 데이터 수집 고지, 기본 고객지원 채널이 준비되어 있다.
- 첫 유료 상품 1개가 진화 성능이 아니라 감성 확장으로 제공된다.
- 첫 유료 상품의 구매/복원/권한 반영이 정상 동작한다.
- 첫 BM 결과로 "계속 만들 가치가 있는가"에 대한 판단 자료가 생긴다.

## 단계 구분

### Phase 0: 로컬 감정 루프 프로토타입

목표는 재미의 씨앗을 확인하는 것이다.

포함한다.

- 온보딩
- 첫 씨앗 생성과 완료
- egg -> baby 부화
- 오늘 화면
- 펫방
- 상태 기반 대사
- SQLite local-first 저장
- 점수/진화 순수 로직 테스트

포함하지 않는다.

- 계정
- 클라우드 동기화
- 실제 결제
- 푸시
- 방 꾸미기 풀기능

### Phase 1: 무료 베타

목표는 실제 사용자가 다시 여는지 확인하는 것이다.

포함한다.

- iOS TestFlight 또는 Google Play Internal Testing 배포
- 기본 분석 이벤트
- crash/error 추적
- 로컬 푸시 또는 오늘의 씨앗 알림
- 첫 성장기 진화
- 기록의 방
- 복귀 경험
- 최소한의 개인정보 처리방침과 지원 채널

성공 질문.

- 사용자가 설명 없이 첫 씨앗을 만들고 완료하는가?
- 펫방을 자발적으로 열어보는가?
- 말 걸기를 눌러보는가?
- 첫 진화까지 가고 싶어 하는가?
- 미완료나 복귀 상황에서 죄책감보다 다시 시작하고 싶은 감정이 드는가?

### Phase 2: 무료 공개 베타

목표는 작은 규모의 공개 사용자에게 제품 메시지와 안정성을 검증하는 것이다.

포함한다.

- 스토어 공개 또는 제한 공개
- 앱 아이콘, 스크린샷, 설명문
- 온보딩 퍼널 개선
- 주요 기기 QA
- 데이터 migration 안정화
- 사용자 피드백 수집 루프

아직 보류한다.

- 여러 유료 상품
- 구독
- 시즌 패스
- 랜덤/가챠
- AI 자유 채팅
- 소셜/랭킹

### Phase 3: 첫 BM 검증

목표는 "이 감성 확장에 돈을 낼 사람이 있는가"를 작게 확인하는 것이다.

첫 BM 후보는 다음 중 하나만 고른다.

| 후보 | 추천도 | 이유 |
| --- | --- | --- |
| 비 오는 공부방 테마 | 높음 | 진화 성능에 영향이 없고 펫방 애착을 강화한다 |
| 별솜 새 여정 팩 | 높음 | 백서의 BM 방향과 맞고 새 여정과 자연스럽게 연결된다 |
| 솜 리본/작은 가방 | 중간 | 제작량은 작지만 단독 구매 매력은 약할 수 있다 |
| 프리미엄 진화 연출 | 중간 | 윤리적으로 가능하지만 무료 진화가 초라해지면 안 된다 |

첫 BM에서는 다음을 절대 하지 않는다.

- 진화 속도 판매
- 결 점수 판매
- 새 여정 시작권 판매
- 리셋권 판매
- 유료 랜덤 알 또는 가챠
- 한정 압박 문구

## 첫 BM 추천

1차 MVP의 첫 BM은 `비 오는 공부방 테마`를 추천한다.

이유는 다음과 같다.

- 무료 핵심 진화 경험을 건드리지 않는다.
- 제작 범위가 펫 1종보다 작고 명확하다.
- TodoSom의 감성 가치와 잘 맞는다.
- 구매하지 않아도 앱의 핵심 기능이 부족해 보이지 않는다.
- 펫방 방문률이 높을수록 구매 의도를 자연스럽게 판단할 수 있다.

대안으로는 `별솜 새 여정 팩`이 좋다. 다만 새 유년기 펫은 idle/talk/happy/eat/sleep/evolve 등 상태 아트 제작량이 커지므로 첫 BM으로는 방 테마보다 부담이 크다.

## 1차 MVP에 필요한 스택

| 영역 | 스택 | 도입 시점 |
| --- | --- | --- |
| 앱 | React Native + Expo | Phase 0 |
| 언어 | TypeScript | Phase 0 |
| 라우팅 | Expo Router | Phase 0 |
| 로컬 DB | expo-sqlite | Phase 0 |
| 상태 | Zustand | Phase 0 |
| 테스트 | Vitest 또는 Jest | Phase 0 |
| 빌드/배포 | EAS Build | Phase 1 |
| 분석 | 선택한 analytics SDK 또는 가벼운 이벤트 레이어 | Phase 1 |
| 오류 추적 | Sentry 등 | Phase 1 |
| 푸시 | Expo Notifications | Phase 1 |
| 계정/동기화 | Supabase Auth + RLS | Phase 2 또는 BM 전후 |
| 결제 | RevenueCat | Phase 3 |

## 1차 MVP의 핵심 원칙

- 백서는 버리지 않는다. 다만 첫 실행 범위와 미래 그림을 구분한다.
- 무료 경험이 먼저 사랑받아야 한다.
- 첫 BM은 돈을 내면 더 강해지는 것이 아니라 더 애틋하고 예뻐지는 방향이어야 한다.
- 1인 창업자는 기능 수가 아니라 반복 가능한 QA와 작고 선명한 제품 경험으로 이긴다.
- "귀여운 앱"이 아니라 "다시 시작하게 해주는 앱"이라는 정체성을 끝까지 지킨다.
